When would you want to use a remote repository rather than keeping all your work local?
Cuando sea un proyecto de una empresa o si estoy trabajando en equipo o se desean permitir contribuciones.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
Si hay varios desarrolladores en el equipo probablemente podria estar descargando mas de lo usual.


Describe the differences between forks, clones, and branches. When would you use one instead of another?
Fork cuando quiero trabajar sobre un proyecto existente de otro desarrollador del cual no soy colaborador.
Clone Copiar un repo a mi maquina local sabiendo que no puedo hacer push al codigo o empezar con un reposiorio en blanco
Branch Trabajar en una nueva feature o arreglar un bug.

What is the benefit of having a copy of the last known state of the remote stored locally?

Es menos probable que tengamos conflictos al pushear nuestros cambios.

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?
Unificar el codigo seria dificil sin github, quiza si usamos algo como google docs en el futuro seria mas facil.

When would you want to make changes in a separate branch rather than directly in master? What benefits does each approach have?
I want to use a separate branch to have master working all the time. Merge the code directly in master could be easy to manage but also we could find scenarios where master is not working.
